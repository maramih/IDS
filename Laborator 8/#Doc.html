LOGISTIC REGRESSION

Regresia logistica este o metoda de analiza multivariata a datelor de marketing din ce in ce mai des utilizata datorita faptului ca numarul conditiilor ce trebuie sa fie indeplinite este mai redus decat, de exemplu, in cazul analizei discriminante.

Regresia logistica modeleaza relatia dintre o multime de variabile independente xi (categoriale, continue) si o variabila dependenta dihotomica (nominala, binara) Y. Spre deosebire de regresia clasica, aici una sau mai multe variabile predicitive/explicative (variabile independente) pot fi categoriale, obligatoriu variabila dependenta, deci in cazul acestui nou model este vorba de caracterul nenumeric, calitativ (categorial) al unora dintre variabilele sale.

Modelul regresiv logistic va estima proportia subiectilor care vor avea aceeasi caracteristica de interes pentru stadiul statistic, sau echivalent, probabilitatea ca un subiect oarecare din populatie sa aiba o anumita caracteristica.

Implementarea regresiei logistice din scikit-learn poate fi accesata din clasa LogisticRegression, iar ea poate face fit binar, Unul vs Restul(One-vs-Rest) sau multinomial, cu doua optiuni de regularizare, cu scopul de a minimiza functia de cost.

Solverii implementati in aceasta clasa sunt: "liblinear", "newton-cg","lbfgs", "sag" si "saga".

*liblinear - utilizeaza algoritmul CD (coordinate descent), insa acesta nu poate invata un adevarat model multinomial, problema fiind optimizata cu ajutorul metodei One vs Rest pentru a putea fi antrenat pe bucati binare pentru toate clasele.

*sag - utilizeaza Stochastin Average Gradient Descent; este mai rapid pentru un set mare de date, cand sunt atat numarul de sample-uri,cat si cel de atribute este destul de mare.

*lbfgs - este un algoritm de optimizare care aproximeaza algoritmul BroydenFletcherGoldfarbShanno; recomandat pentru un set redus de date, insa performanta sa are de suferit pentru seturi mai largi.

RIDGE REGRESSION

Ridge Regression este o tehnic pentru analiza datelor de regresie multiple care sufer de multicoliniaritate. Cand se ntmpl multicoliniaritatea, estimrile celor mai mici ptrate sunt impariale, dar variaiile lor sunt mari, astfel nct acestea pot fi departe de adevrata lor valoare. Prin adugarea unui grad de prtinire la estimrile de regresie, ridge regression reduce erorile standard. Se sper ca efectul final va fi acela de a oferi estimri mai fiabile.

Unul dintre principalele obstacole n utilizarea Ridge Regression este alegerea unei valori adecvate de k. Hoerl i Kennard (1970), inventatorii Ridge Regression, au sugerat utilizarea unui grafic pe care l-au numit Ridge Trace. Acest plot arat coeficienii de Ridge Regression in funcie de k. La vizualizarea Ridge Trace, analistul alege o valoare pentru k pentru care coeficienii de regresie s-au stabilizat. Adesea, coeficienii de regresie vor varia foarte mult pentru valori mici ale k i apoi se vor stabiliza. Alegand cea mai mic valoare a lui k posibil (care introduce cel mai mic bias), coeficienii de regresie par s rmn constani. Deci cretereac lui k va conduce n cele din urm coeficienii de regresie la zero. Un exemplu de Ridge Trace este mai jos:

Totodata, implementarea Ridge Regression poate fi accesata din clasa Ridge, iar solverii implementati in aceasta sunt:

*'Svd' - folosete o descompunere a valorii singulare a lui X pentru a calcula coeficienii Ridge. Mai stabil pentru matricile singulare dect "cholesky".

*"Cholesky" - folosete funcia standard scipy.linalg.solve pentru a obine o soluie n form nchis.

*"Sparse_cg" - utilizeaz solverul conjugat gradient aa cum se gsete n scipy.sparse.linalg.cg. Ca algoritm iterativ, acest solver este mai adecvat dect "cholesky" pentru date la scar larg (posibilitatea de a seta tol i max_iter).

*'Lsqr' - utilizeaz rutina dedicat celui mai mic ptrat regularizat scipy.sparse.linalg.lsqr. Este cea mai rapid i folosete o procedur iterativ.

*"Sag"- folosete o coborre medie a gradientului stochastic, iar "saga" folosete versiunea sa mbuntit, imparial numit SAGA. Ambele metode utilizeaz o procedur iterativ i sunt adesea mai rapide dect ali solveri atunci cnd att n_samples, ct i n_features sunt mari.

Convergena rapid "sag" i "saga" este garantat doar pentru caracteristicile cu aproximativ aceeai scar. Putei pre-prelucra datele cu ajutorul unui scalar din sklearn.preprocessing.

Ultimii cinci solveri susin att date complexe, ct i mai simple. Cu toate acestea, numai "sag" i "saga" accept intrari mai simple atunci cnd fit_intercept este Adevrat.

Bayesian Regression

Bayesian Ridge estimeaz un model probabilistic al probleme de regresie. Valoarea pentru parametru w este data de un Gaussian sferic: Valorile pentru alpha i lambda sunt alese pentru a fi distribuii gamma, conjugatele anterioare pentru precizia Gaussianei.

Modelul rezultat se numete regresie Bayesian Ridge i este similar cu Ridge clasic. Parametrii w, alpha si lambda sunt estimati n comun n timp ce se face fit-ul modelului. Hiperparametrii rmasi sunt parametrii gamma anteriori peste alpha i lambda. Acestea sunt de obicei alese s nu fie informative. Parametrii sunt estimai prin maximizarea probabilitii logice marginale.

n comparaie cu estimatorul OLS, valorile coeficientului w sunt uor deplasate ctre zerouri, ceea ce le stabilizeaz.

Deoarece precedentele valori ale lui w sunt valori Gaussiene, histograma coeficientilor w estimati este Gaussiana.

Estimarea modelului se face prin maximizarea iterativ a probabilitii logice marginale.

De asemenea, comparam prediciile i incertitudinile pentru regresia Ridge Bayesian pentru o regresie dimensional, utiliznd o extindere a caracteristicilor polinomiale. Incertitudinea ncepe s urce pe partea dreapt a graficului. Acest lucru se datoreaz faptului c aceste eantioane de testare sunt n afara gamei de probe de antrenament.

SGD Regressor

SGD sau Stochastic Gradient Descent: gradient-ul pierderilor este estimat la fiecare sample la un moment dat si este actualizat de-a lungul rularii cu o rata de invatare (learning rate).

Regularizatorul are sarcina de a micsora parametrii modelului cat mai apropiat de 0, utilizand fie norma euclidiana patrata L2, fie norma absoluta L1, fie o combinatie intre cele doua (Elastic Net) pentru a regulariza functia de loss.

Daca actualizarea parametrilor depaseste valoarea 0.0 datorita regularizatorului, actualizarea este trunchiata la 0.0 pentru a permite invatarea modelelor slab populate si pentru a obtine selectia caracteristicilor.
